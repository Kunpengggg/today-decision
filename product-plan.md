# 决策运势产品规划与市场分析

## 1. 一句话定位

一个面向中文用户的「轻决策陪伴工具」：用户在犹豫要不要做某件事时，产品结合星座、生肖/八字元素、今日/月度/年度节奏，给出「适合做 / 建议缓做 / 不建议做」的行动建议，并提供简短理由与可展开分析。

建议避免把产品包装成“算命决定人生”，更适合表达为：

- 传统文化灵感 + 情绪安抚
- 决策提示 + 风险提醒
- 日常仪式感 + 分享型内容

## 2. 市场判断

### 需求成立

2025 年年轻人玄学消费报告显示，年轻用户对玄学的需求已经日常化、内容化、社交化。样本中，事业运、财运是最高频需求，八字命理、星座、塔罗是主要关注方向；很多用户把玄学当作缓解焦虑、获得解释感和行动勇气的方式。

这说明你的产品核心需求不是“用户真的需要一个绝对答案”，而是：

- 拿不定主意时，需要外部提示。
- 焦虑时，需要一个温和解释。
- 做决定前，需要降低心理负担。
- 好玩、有仪式感、能截图分享。

### 竞品态势

市场上已有星座运势、塔罗、八字排盘、AI 算命等产品，但多数问题是：

- 内容偏泛，和用户当下具体问题连接弱。
- 重报告，轻决策。
- 重付费测算，轻日常留存。
- 内容“玄”但产品体验旧。

可切入的差异化方向是「具体问题决策」：不是只告诉用户“今日财运三星”，而是回答“今天要不要提涨薪”“这个月要不要换工作”“今年适不适合创业”。

## 3. 合规与表达边界

中国大陆环境下，直接以“算命、占卜、预测吉凶、保证转运”等方式宣传和收费，存在明显审核与合规风险。更稳妥的表达是：

- 用「传统文化参考」「情绪陪伴」「个人反思」「娱乐参考」替代「算命预测」。
- 明确结果不构成医疗、法律、投资、婚恋等重大决策建议。
- 对高风险问题触发安全提示，例如投资、借贷、疾病、离婚、辞职等。
- 不承诺准确率，不制造恐吓，不诱导高价付费化解。

产品内的判断文案建议使用「倾向」「建议」「提醒」「可以考虑」等表达，而不是「必然」「注定」「一定」。

## 4. MVP 功能规划

### 第一阶段：可上线验证版

目标：验证用户是否愿意每天打开，并愿意针对具体问题使用。

功能：

- 用户档案：昵称、生日、出生时辰、性别可选、偏好模式。
- 运势视角：今日、本月、本年。
- 模式选择：星座、生肖/八字灵感、综合。
- 决策输入：用户输入“我要不要……”
- 决策输出：做 / 缓做 / 不做。
- 简短理由：3 句话以内。
- 展开分析：机会、风险、行动建议、提醒。
- 本地历史：保存最近 20 条决策。
- 分享卡片：后续做成图片或小程序分享卡。

### 第二阶段：留存与传播

- 每日提醒：今日适合处理的事、今日避免的事。
- 每周/月复盘：用户做过哪些决策，结果如何。
- 情绪标签：焦虑、期待、犹豫、生气、疲惫。
- 分享文案：朋友圈/小红书风格生成。
- 主题内容：五行穿衣、幸运色、行动窗口。

### 第三阶段：商业化

- 会员：无限决策、历史复盘、详细报告、年度趋势。
- 单次付费：深度分析、关系/事业/财务专题。
- 内容电商：手账模板、幸运色穿搭、传统文化周边。
- B 端合作：心理陪伴、泛娱乐社区、情侣/职场测试组件。

## 5. 技术路线

### localStorage 可以用，但只适合 MVP

你的 localStorage 想法是对的，第一阶段可以不用数据库，优点是：

- 开发快，成本低。
- 无登录也能跑。
- 适合验证核心体验。
- 隐私压力相对小，因为数据留在用户设备。

但它不能支撑长期商业化：

- 换设备数据丢失。
- 无法做会员、支付权益、召回、A/B 测试。
- 无法做用户生命周期分析。
- 小程序本地缓存通常有容量限制，且可能被系统清理。
- 出生时间属于敏感个人信息，正式上线仍需隐私政策和最小化采集。

建议路线：

1. Web 原型：localStorage。
2. 小程序 MVP：wx.setStorage + 可选云开发。
3. 商业化版：账号体系 + 服务端数据库 + 支付 + 内容生成服务。

## 6. MVP 判断规则

初版不需要真的做完整命理引擎，可以用可解释的规则系统：

- 星座：按生日映射星座，结合日期生成日/月/年节奏。
- 八字灵感：先用生肖、季节、时辰、五行倾向做轻量解释。
- 决策类型：从用户问题中识别事业、感情、金钱、健康、人际、学习。
- 输出：根据「用户模式 + 时间范围 + 决策类型 + 当天日期种子」生成稳定但有变化的建议。

这比一开始追求完整八字排盘更实际。等验证用户愿意用，再接入专业历法库和更严谨的排盘。

## 7. 关键指标

- 激活率：首次录入信息后完成一次决策的比例。
- 次日留存：第二天是否回来查看今日运势。
- 决策频次：每人每周输入几次问题。
- 展开率：用户是否点击详细分析。
- 分享率：是否愿意分享结果卡片。
- 付费转化：深度分析或会员转化率。

## 8. 当前结论

可行，但不要做成传统“算命 App”。更好的机会是做一个「玄学外衣下的轻决策与情绪陪伴产品」。先用 localStorage 做网页原型验证体验，然后转微信小程序做传播，再根据数据决定是否接数据库和支付。
